## 環境建置
本專案結合前端畫面呈現與後端資料庫資料讀取兩部分，因此請連同node (https://github.com/Kris0201/node) 此專案一併下載，並視需求開啟mySQL資料庫，方能正常使用，詳細開啟專案方式如下：

1. 將本專案與上述提供之node專案一同下載到您的電腦中
react:  <https://github.com/Kris0201/react-topic.git>
node： <https://github.com/Kris0201/node.git>

2. 下載好後以vs code(Visual Studio Code)編輯器開啟，並分別於兩個專案的終端機輸入指令
`npm i`


3. 執行完安裝後，若欲看到完整資料庫資料，可至node專案中的"data"資料夾中下載所有sql檔；若不需要安裝資料庫，可直接進行步驟4
![image](https://github.com/Kris0201/react-topic/blob/master/%E8%AA%AA%E6%98%8E%E5%9C%96%E7%89%87/data-sql.png)

4. 在react專案終端機輸入yarn start指令，在node專案終端機輸入nodemon指令即可開啟靜態頁面。注意，若無建置資料庫，部分頁面(例如「商品列表」與「商品詳細頁」圖片或訊息將無法正常顯示，以下將於說明中放入圖片以便參考。


## 專案介紹
本專案為多人協作作品，非一己之力完成。我個人主要負責頁面為「商品列表」與「商品詳細頁」，詳細小組分工置於文末，敬請參考。

### 專案名稱: Drunken Cake || 醉糕品味

**專案主題**
本專案是以商品販售為主的網站設計，網站主要由「首頁」、「商品」、「體驗」、「場地租借」和「關於我們」組成。

主要的販售商品為結合酒精與蛋糕的商品，同時亦售有其它體驗型商品，例如在「體驗」頁面中的各式諸如手作課程、職人講座。此外，也提供蛋糕教室租借的服務。

**專案使用技術**

**各頁面功能說明**

- 首頁：以沉穩的巧克力色系為基底，搭配金色字體與淺色背景色，營造網頁質感以及食物的美味感受，試圖在暗色與亮色間取得平衡，並以區塊分明的方式排放各區資訊。

+ 商品
    + 商品列表：頁面先以一橫幅滿版背景圖開頭，背景圖主題明確，點明商品的特色：酒與蛋糕。內容面的排版方面，選用深色背景的圖片襯托出商店欲呈現的「時尚」與「高級」的感受，並適度留白以避免過多的暗色調造成距離感和壓力。在技術運用方面，顧客可在商品列表頁直接輸入關鍵字搜尋商品，或點選分類標籤以搜尋相關產品。
    + 商品詳細頁：商品詳細頁則加入了「加入收藏」的功能，點選後就可在會員中心的「我的收藏」看到收藏的商品。最後，頁面的下半部會有一個輪播圖片區，供顧客參考其它不同的產品。

+ 體驗
    + 體驗主頁：在活動體驗的主頁，使用者可透過搜尋功能尋找各式活動體驗，頁面下方放置熱門活動介紹輪播牆，可直接點入觀看。
    + 搜尋結果頁：本頁顯示搜尋結果與數量，左方放置分類列表，使用者可以依照分類查看活動體驗項目。亦可透過價格篩選與商品排序，讓使用者可以更精準的篩選到自己想要的活動體驗。
    + 體驗詳細頁：本頁上方為商品輪播牆，使整個頁面變得活潑生動。活動體驗的內容描述則是以線段分隔帶出簡潔有力之感，降低使用者閱讀的負擔，「加入購物車」按鈕則使用金色底，並顯眼地放置在網頁右側，使用者可以輕鬆地購買喜愛的活動體驗。

- 場地租借：進入教室租借頁面，使用者可透過搜尋選取出想預約的日期以及人數，搜尋後下方顯現出還未被預約的教室。進入預約頁面後，右邊為詳細教室資訊，左邊可以選取自己想預約的時段、人數與日期，選取完畢即出現價格，點擊預約按鈕就能加入購物車等待付款。

+ 購物車
    + 彈出式購物車：點選導覽列中的購物車後即可彈出小型視窗，顯示目前有哪些商品已經被加入了購物車，以及總金額是多少，讓使用者能夠方便快速地掌握資訊。
    + 結帳頁面：購物車(結帳頁面)分為三個頁面，按結帳流程依序為商品清單、配送與付款資訊、訂購完成，詳述如下：
    1. 商品清單：本頁顯示使用者加入購物車內商品的品項、單價以及數量，並提供更改數量、移除商品以及金額小計等功能。
    2. 配送與付款資訊：本頁是讓使用者選擇配送方式、填寫配送資訊並完成結帳。使用者選擇配送方式後，頁面會顯示相關運費，並將運費與商品小計金額加總，得到訂單總價格。接著使用者填寫配送資訊，如訂購人姓名、收件人姓名、收件地址等等。最後使用者選擇付款方式，並填寫相關資訊後，完成結帳。
    3. 訂購完成：本頁顯示感謝詞，以及訂單詳細資訊，包含購買品項、配送與付款資訊。
